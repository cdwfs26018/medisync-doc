# Product Backlog - MediSync

## Description
This backlog contains the main features of the MediSync application.

---

## User Stories

### Must Have

- **US-01**
  As a doctor, I want to share an ordonnance with a specialist so that they can follow the patient treatment.  
  **Priority:** Must Have  
  **Story Points:** 8

- **US-02**
  As a doctor, I want to search a patient by name so that I can quickly access their file.  
  **Priority:** Must Have  
  **Story Points:** 3

- **US-03**
  As a doctor, I want to receive a notification when a new ordonnance is shared so that I stay informed.  
  **Priority:** Must Have  
  **Story Points:** 8

---

### Should Have

- **US-04**
  As an admin, I want to export users list in CSV so that I can analyze data.  
  **Priority:** Should Have  
  **Story Points:** 5

- **US-05**
  As a patient, I want to download my ordonnance in PDF so that I can keep it.  
  **Priority:** Should Have  
  **Story Points:** 5

---

### Could Have

- **US-06**
  As a doctor, I want a dashboard with statistics so that I can monitor activity.  
  **Priority:** Could Have  
  **Story Points:** 8

---

## Notes

- Priorities follow MoSCoW method
- Story Points are estimated using Fibonacci scale
