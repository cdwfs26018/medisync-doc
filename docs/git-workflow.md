# Git Workflow

## Branching strategy

We use GitHub Flow:
- Create branch from main
- Commit changes
- Push branch
- Open Pull Request
- Merge after review
- Delete branch

## Commit rules

- Messages in English
- Maximum 10 words
- Use action verbs

## Network constraint

Port 22 is blocked, so we use HTTPS:

git clone https://github.com/username/repo.git
